# Analysis: DAL Admin API CRUD Endpoints

## 1. Executive Summary

The plugin registers two entity definitions (`tdsa_search_log` and `tdsa_search_stats`) via the `shopware.entity.definition` tag. Shopware's Admin API automatically generates **10 endpoints** (5 CRUD operations × 2 entities) from these definitions, plus advanced search endpoints and the bulk `/api/_action/sync` endpoint. No manual controller code is needed for standard CRUD.

## 2. Scope

All automatically generated Admin API endpoints derived from the two `EntityDefinition` classes:
- `src/Entity/SearchLog/SearchLogEntityDefinition.php`
- `src/Entity/SearchStats/SearchStatsEntityDefinition.php`

Registered in `src/Resources/config/services.xml` with the `shopware.entity.definition` tag.

## 3. Methodology

Code review of the `EntityDefinition` classes and Shopware 6.7 Admin API conventions. Endpoint paths follow the pattern `/api/{entity-name}` where underscores are converted to hyphens (kebab-case).

## 4. Findings

## 5. Options & Alternatives

N/A — these are auto-generated by Shopware's framework; no alternative is applicable.

## 6. Recommendations

No action needed. The DAL-generated endpoints are fully functional. Notes for consumers:

- **Upsert via sync:** For bulk importing search log data, use `POST /api/_action/sync` with an `upsert` operation on `tdsa_search_log`.
- **ACL privileges:** Admin modules already reference `system.zero_search.viewer`. Ensure API access is gated behind appropriate ACL privileges when restricting access.

## 7. Risks & Trade-offs

- **No soft-delete:** Neither entity definition defines a `DeletedAtField`, so delete operations are hard deletes.
- **No write-protection:** By default, the Admin API allows writes to these entities unless ACL privileges restrict them.
- **Data growth:** `tdsa_search_log` receives a row per storefront search. Without governance, the listing endpoint could become slow at scale (mitigated by the consolidation task).

## 8. Next Steps

1. Ensure admin administration modules use these endpoints correctly (already done via `searchApiService` in the Vue admin components).
2. Verify ACL privilege mapping for the two entities matches the admin module setup.
3. Consider adding `since`/`until` date filters to the admin listing components for large datasets.
